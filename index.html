Overview

Muilessium is the UI framework or rather a collection of UI components and helpers for developing responsive and accessible websites. This framework written in LESS and ES6 and can be included in a project as is or can be easily customized for your needs. Muilessium supports WAI-ARIA attributes and generates most of them automatically.

See on GitHub

Grid

Muilessium uses a grid with 12 columns based on flexbox (and has legacy version based on floats, but only in custom builds). Rows and columns have classes .mui-row and .mui-column and columns can have options -s* for a small screen, -m* for a medium screen and -l* for a large screen. Default breakpoints are 400, 600 and 1200px. See src/less/components/grid-flex.less for more information.

Page layout

Basic page layout includes header, page container and footer. This layout is the simplest way to create a sticky footer, but IE11 does not support flexbox greatly, so we have some incorrect behavior in it.

    <header class='mui-page-header'>header</header>
    <div class='mui-page-container'>...</div>
    <footer class='mui-page-footer'>footer</footer>
¶

Recommended viewport

Muilessium uses Hammer.js and it is recommended to use the following meta in your pages:

    <meta name='viewport' content='user-scalable=no, width=device-width, initial-scale=1, maximum-scale=1'>

Utils

Muilessium exports a number of internal utilities as window.Muilessium.Utils including

  • Ajax utilities
  • WAI-ARIA utilities
  • Checking for elements in page
  • Manipulating with HTML attributes
  • Manipulating with CSS classes
  • Manipulating with Focus & Click events
  • Scroll utilities (smooth scrolling, scrollFire)
  • Viewport utilities
  • Uncategorized utilities (debounce, callOnce, extend e.t.c)

See utils docs for more information

F.A.Q.

Can I use Muilessium in my project?
Yes, Muilessium is distributed under the MIT license, so You can use it in Your projects.
What browsers are supported?
All modern browsers, including IE11.
Who is the author?
My name is Ivan Bogachev. I'm a musician and a front-end developer, You could see some of my pens on codepen previously. You can visit my website for more information.
What can I do if I found a bug?
If you found a bug - don't worry, Muilessium is is pre-release state and can have some bugs, You can open an issue on GitHub or send me an email and I will fix a problem when I will have a time.
I have some suggestions for improvement of Muilessium
That's great! You can fork Muilesssium, add Your functionality and create a pull request to the dev branch, or contact with me by email or twitter (but I should notify You - English is not my native language, maybe I'll need some time to translate messages)
